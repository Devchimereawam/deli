## Before connecting to WhatsApp

There is **one important piece missing.**

Right now your webhook **returns the reply string**, but it **does not send a message back to WhatsApp**.

Currently your flow is basically:

```
WhatsApp

↓

Webhook

↓

Process

↓

return "Welcome"

↓

Done
```

Meta **doesn't read that response and send it to the user.**

Instead you must call the **WhatsApp Cloud API Send Message endpoint**.

So before testing on WhatsApp, we'll add outbound messaging.

## File

```
whatsapp/client.py
```

```python
import os

import requests


ACCESS_TOKEN = os.getenv("WHATSAPP_ACCESS_TOKEN")

PHONE_NUMBER_ID = os.getenv(
    "WHATSAPP_PHONE_NUMBER_ID"
)


class WhatsAppClient:

    URL = (
        f"https://graph.facebook.com/v23.0/"
        f"{PHONE_NUMBER_ID}/messages"
    )

    def send_text(
        self,
        phone,
        message,
    ):

        headers = {

            "Authorization": f"Bearer {ACCESS_TOKEN}",

            "Content-Type": "application/json",

        }

        payload = {

            "messaging_product": "whatsapp",

            "to": phone,

            "type": "text",

            "text": {

                "body": message

            }

        }

        response = requests.post(

            self.URL,

            headers=headers,

            json=payload,

        )

        print(response.json())

        return response
```

# One improvement before moving to restaurants

I recommend we **don't jump to restaurants immediately after this**.

Instead, we should first implement the complete **WhatsApp home navigation**, because that's the core user experience. The next milestone should be:

* Option **1** → Browse restaurants (initially returns "No restaurants yet" until we add them).
* Option **2** → Search food.
* Option **3** → My Orders.
* Option **4** → Change Location.

This gives us a fully interactive WhatsApp application before we start adding restaurant data, making it easier to test the conversation flow end-to-end as we build.
